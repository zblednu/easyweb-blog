Host your website for free

Want your website to be something more than a dead-weight resting on your pc? Fair enough.
We will use Github-Pages functionality to host our blog.

Setting up Github account

If you don't have one, go to https://github.com/ and create an account.

We need a way for your pc to identify itself to your Github. That's what SSH-keys are for. Run the command below in your terminal: ssh-keygen -t ed25519 -N "" -f ~/.ssh/id_ed25519.pub "" && cat ~/.ssh/id_ed25519.pub This will generate a pair of private and public key using some fancy algorithm, place it in your user's home directory and print out the created public key.

Copy the last line of output(the one that starts with 'ssh-ed25519'), go to https://github.com/settings/ssh/new and paste it in the key field.
Name it however you like, then press "Add SSH Key".

Uploading your website to Github

Go to https://github.com/new to create a repo for your blog project. Name it somewhat short, make sure the visibility is set to Public, if you set it to Private other people won't be able you see your website online. Then click create repository and copy the ssh link it generated. Now, back to your terminal, we'll upload your blog to the newly created repository.

  1. cd into your blog's directory and make it a git directory
    2. cd path-to-your-blog
    git init
  2. Switch branch to main and add your entire project as the first commit
    3. git branch -M main
    git add .
    git commit -m "initial commit"
  3. Set the pass to your github repo and upload files
    4. git remote add origin path to your github repo you copied in step 1
    git push -u origin main
Now you should be able to see your blog's files uploaded to your github.

Configuring Github-Pages

Then, on your repository page:

  1. Go to your repository's "Settings" tab
  2. Scroll down to the "Pages" section.
  3. Under "Source", select the "main" branch and the root folder.
  4. Click "Save"
Your website will be live in a minute or two at https://username.github.io/repo-name
Neat, right? From now on GitHub will serve the latest version of index.html located in the repository. So no need to re-configure it every time. Just make sure you've pushed your changes.
