Host your website for free

Want your website to be something more than a deadweight resting on your pc?
In this article you will configure Github Pages to host your blog.

Setting up Github account

If you don't already have one, go to github.com and create an account.

We need a way for your PC to identify itself to your Github. That's what SSH-keys are for.
Run the command below in your terminal: ssh-keygen -t ed25519 -N "" -f ~/.ssh/id_ed25519 && cat ~/.ssh/id_ed25519.pub It generates a pair of public/private keys using some fancy algorithm, places it in your user's home directory and prints out the created public key.

Copy the last line of terminal's output (the one that starts with "ssh-ed25519").
Go to github.com/settings/ssh/new and paste it in the "Key" field. Name it however you like, then press "Add SSH Key".

Uploading your website to Github

Go to github.com/new to create a repo for your blog project.

! Make sure the visibility is set to Public, if you set it to Private other people won't be able you see your website online.

Then click "Create repository" and copy the ssh link it generated.

Now, back to your terminal, we'll upload your blog to the newly created repository.

! Make sure that file of the landing page for your website is named "index.html". Standard behaviour for any web-server is to try to serve "index.html" when no resource is specified in the web request.
  1. cd into your blog's directory, make it a git directory and set the path to your GitHub: cd path-to-your-blog
    git init
    git remote add origin ssh-path-to-your-github-repo
  2. Switch branch to main and add your entire project as the first commit: git branch -M main
    git add .
    git commit -m "initial commit"
  3. Finally, upload the files: git push -u origin main

You should be able to see your blog's files uploaded to your GitHub repository.

Configuring Github Pages

On your repository page:

  1. Go to repository "Settings" tab
  2. Scroll down to the "Pages" section
  3. Under "Source", select "Deploy from a branch"
  4. Select "main" branch and the root folder
  5. Click "Save"

In the end, it should look like so:

After a short period of time (shouldn't take more than 5 minutes), your site will be live at:
username.github.io/repo-name

Or, simply reload the "Pages" tab and you'll see the link:

From now on, GitHub will serve the latest content in your repository, so there's no need to reconfigure it every time you make a change to your website.
Just make sure you've pushed your changes, and the file named "index.html" is present.
